[v2.0]
	Help screen (Left Thumb to toggle)

	Check_Usable ini option (only show trainers for games you have)

	IGK combo to reload dashboard from launcher (back + black)

	DVD gfx displayed on screen when a title is initialized in tray

	SoundTrack support (Plays soundtracks from hdd saved w/ stock dash)
		(default defined in ini and random play available)
		
	MOD, XMs, S3M File playback
		
	Previous position remembered when stepping backwards through scenes

	Full Skinning Support (see included examples)

	GamePad B button added for scene backwards

	Launch title without trainer

	Multiple Game_Path's (20 max)

	Support for next generation trainer format "xbtf"
	
	X3/SmartXX/Aladin4 LCD Support enable / disable in ini file
	
	Debug.log file to help track down problems

	Delete trainers from hdd (IGK)
	
[CHANGED]
	New scene order:
		(Games -> Trainers -> Trainer Options -> Launch)

[FIXED]
	Kernel patch routine should handle soft boots better now (IGR)
	
	Launcher misbehaving if no gamepads are plugged in on start up
	
	Ini parsing buggy
	
	Launcher should initialize faster now

[v2.1]
	Improved kernel patch routine (also attempts to unhook Evox TSR)
	
	Play_Mod_File no longer get sets to TRUE if no soundtracks are found
	
	New partition handling and xbe launching routines
	
	Fixed problem with DVD swapping not being detected 
	
	Launcher now runs at screen resolution 720x480 for nicer looking skins
	
	Added Skip_Intro (you must watch it once before this will work)
	
	6 sided cube in intro now skin able (good if you have children)
	
[V2.2]
	Added support for MenuX based multi-game DVD's
	
	Removed intro completely (it was getting old)
	
	Added Alternate_Launch_Method to ini (use v2.0 launch method)
	
	You can now really enjoy the silence (Play_Sound_Tracks added)
	
	Launcher now scrolls extra details from etm trainer's info section 
	
	Ground up re-coding how DVD games are handled (much smoother now)
	
	Increased maximum trainer options to 200 (+99 (107)? props!)
	
	Quick-launch (hit start button to launch selected game with proper 
		trainer and your last used options)
		
	
	Added ini entries to enable/disable mounting of F/G partitions
		(requested by k_away)
		
	Increased ETM buffer limit to match xbtf (can now run approx 7x larger
		ETM trainers then the Evolution-X dash!)
	
	Kernel hack now uses Xbox's LED to show xbe execution status!	
		Xbox LED GREEN means trainer in memory is being checked against
			xbe about to be executed (good for people who's led's are 
			red by default like mine)
			
		Xbox LED RED means trainer in memory did not match xbe just executed
	
		Xbox LED ORANGE means trainer in memory matched and was applied to xbe
		just executed
		
		Keep in mind that kernel hacks stay in memory until a cold boot :)
		

	Several misc tweaks / bug fixes
